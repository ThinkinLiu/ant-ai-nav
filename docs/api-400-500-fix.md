# API 400/500 错误修复指南

## 问题描述

部署后访问网站时，所有 API 调用返回 400 或 500 错误：

```
GET https://mayiai.mayiai.site/api/home 400 (Bad Request)
GET https://mayiai.mayiai.site/api/tools 400 (Bad Request)
GET https://mayiai.mayiai.site/api/categories 400 (Bad Request)
GET https://mayiai.mayiai.site/api/announcements 400 (Bad Request)
```

## 根本原因

API 路由使用的是**同步版本**的 Supabase 客户端函数 `getSupabaseClient()`，该函数只读取环境变量，不支持运行时配置文件。

当首次部署后：
1. 环境变量可能未配置（或使用占位符）
2. 用户通过 `/settings` 页面配置了数据库
3. 但 API 仍然无法读取运行时配置
4. 导致所有 API 调用失败

## 解决方案

### 方案 1: 使用异步版本的客户端（已实现）

已修改关键 API 路由，使用异步版本的客户端 `getSupabaseClientAsync()`：

- ✅ `/api/home` - 首页数据
- ✅ `/api/tools` - 工具列表
- ✅ `/api/categories` - 分类列表
- ✅ `/api/announcements` - 公告列表

### 方案 2: 重新部署镜像

由于代码已修改，需要重新构建并部署：

```bash
# 1. 停止旧容器
docker stop ant-ai-nav
docker rm ant-ai-nav

# 2. 删除旧镜像
docker rmi ant-ai-nav:latest

# 3. 推送代码到 GitHub
git add .
git commit -m "fix: 修复 API 支持运行时配置"
git push origin main

# 4. 在 GitHub Actions 运行 "Build Docker Image" 工作流

# 5. 下载新的 docker-image.tar.gz

# 6. 加载新镜像
docker load < docker-image.tar.gz

# 7. 启动新容器
docker run -d -p 5000:5000 --name ant-ai-nav ant-ai-nav:latest
```

### 方案 3: 使用环境变量（临时方案）

如果无法立即重新部署，可以使用环境变量配置数据库：

```bash
# 停止容器
docker stop ant-ai-nav
docker rm ant-ai-nav

# 使用环境变量启动
docker run -d \
  -p 5000:5000 \
  --name ant-ai-nav \
  -e NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co \
  -e NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key \
  ant-ai-nav:latest
```

## 技术细节

### 同步客户端（不支持运行时配置）

```typescript
// ❌ 旧方式：同步客户端
import { getSupabaseClient } from '@/storage/database/supabase-client'

const client = getSupabaseClient()
// 只读取环境变量，不支持运行时配置
```

### 异步客户端（支持运行时配置）

```typescript
// ✅ 新方式：异步客户端
import { getSupabaseClientAsync } from '@/storage/database/supabase-client'

const client = await getSupabaseClientAsync()
// 支持环境变量和运行时配置文件
```

### 配置优先级

`getSupabaseClientAsync()` 按以下优先级读取配置：

1. **环境变量**：`NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY`
2. **COZE 环境变量**：`COZE_SUPABASE_URL` / `COZE_SUPABASE_ANON_KEY`
3. **运行时配置文件**：`/app/config/database.json`

## 常见问题

### Q: 为什么不直接修改所有 API 路由？

**A**: 为了快速修复，只修改了最关键的 4 个路由（home, tools, categories, announcements）。其他路由可以后续逐步更新。

### Q: 环境变量和运行时配置哪个优先？

**A**: 环境变量优先级更高。如果环境变量已配置，运行时配置将被忽略。

### Q: 如何确保配置正确？

**A**: 
1. 访问 `/settings` 页面
2. 填写 Supabase URL 和 Anon Key
3. 点击"验证连接"测试
4. 保存配置
5. 刷新页面验证 API

### Q: 修改后还是 400 错误？

**A**: 可能原因：
1. 未重新部署镜像（需要重新构建）
2. 数据库配置未保存
3. 浏览器缓存（硬刷新：Ctrl+Shift+R）
4. 容器未重启
